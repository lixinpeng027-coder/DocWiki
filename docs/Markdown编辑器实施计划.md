# Markdown 编辑器实施计划

## 一、目标

将现有 Markdown 纯文本框升级为接近飞书云文档的本地编辑体验，同时保证：

- 正文最终仍保存为标准 Markdown 文件。
- 图片保存到当前文档同级的 `assets` 文件夹。
- Markdown 中使用相对图片路径，项目移动后引用仍然有效。
- 支持所见即所得与 Markdown 源码两种编辑方式。
- 保存前继续使用文件修改时间进行冲突检测。

## 二、已经完成

### 1. 本地依赖

已安装并本地加载：

- `marked`：Markdown 转 HTML。
- `DOMPurify`：清理渲染后的 HTML。
- `turndown`：HTML 转 Markdown。
- `turndown-plugin-gfm`：支持 GFM 表格等格式转换。

### 2. 编辑器界面

已添加：

- 所见即所得编辑区域。
- Markdown 源码切换按钮。
- 撤销。
- 重做。
- 粗体。
- 斜体。
- 无序列表。
- 插入链接。
- 插入表格。
- 插入图片。

### 3. 表格

已实现：

- 所见即所得模式插入三列表格。
- 表格单元格直接编辑。
- 源码模式插入 GFM Markdown 表格。
- HTML 表格保存时转换为 Markdown 表格。

### 4. 图片

已实现服务端接口：

- 支持 PNG、JPEG、GIF、WebP。
- 单张图片最大 6MB。
- 图片只能写入 `data` 范围内的 `assets` 目录。
- 图片通过本地接口读取，不允许越过 `data` 目录。

已实现前端流程：

- 从编辑器选择图片。
- 自动生成安全文件名。
- 保存到当前 Markdown 同级的 `assets` 文件夹。
- Markdown 中写入 `assets/图片名` 相对路径。
- 所见即所得区域显示本地图片。

### 5. Markdown 转换

已实现：

- Markdown 安全渲染为 HTML。
- HTML 转换回 Markdown。
- 图片接口地址转换回相对 Markdown 路径。
- 字符数按照最终 Markdown 内容计算。

## 四、本阶段验收标准

本阶段完成时，应满足：

1. 用户可以像普通云文档一样直接点击正文编辑。
2. 支持标题层级、基本文字格式、列表、链接、表格和图片。
3. 用户可切换查看或编辑 Markdown 源码。
4. 保存后磁盘中的文件仍是可迁移的标准 Markdown。
5. 图片存放在文档旁的 `assets` 目录并使用相对路径。
6. 重新打开文档后格式、表格和图片能够正确恢复。
