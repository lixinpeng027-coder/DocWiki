# DocWiki 远程更新发布说明

## 工作方式

DocWiki 使用公网 HTTP/HTTPS 更新源，不直接通过 MCP 传输安装包。

更新服务器需要提供两个可下载文件：

1. `latest.json`：版本、安装包地址和校验值。
2. `DocWiki-Setup-x.y.z.exe`：对应版本安装包。

公司电脑在“设置 → 通用设置 → 远程更新源”填写 `latest.json` 的公网地址。保存后，软件每次启动都会自动检查，也可以手动点击“检查更新”。

## 在发布电脑生成清单

完成安装包构建后执行：

```powershell
node scripts/create-update-manifest.mjs `
  "F:\trae_workspace\DocWiki-Setup-1.3.0.exe" `
  "https://updates.example.com/docwiki/DocWiki-Setup-1.3.0.exe" `
  "本次更新说明"
```

命令会在安装包旁生成 `latest.json`，并自动计算 SHA-256。

将安装包和 `latest.json` 上传到公网服务器、对象存储、WebDAV、Gitee/GitHub Release 或其他能直接下载文件的位置。

## latest.json 示例

```json
{
  "version": "1.3.0",
  "url": "https://updates.example.com/docwiki/DocWiki-Setup-1.3.0.exe",
  "sha256": "64位SHA-256校验值",
  "notes": "本次更新说明"
}
```

`url` 也可以写成相对于 `latest.json` 的地址，例如 `./DocWiki-Setup-1.3.0.exe`。

## 数据保护

开始安装前，DocWiki 会把以下内容临时备份到安装目录之外：

- 安装目录下的 `data` 知识库。
- 安装目录下的 `.docwiki\state`，包括模型配置和密钥数据库。

新版本首次启动时会自动恢复并合并这些数据，确认恢复成功后再删除临时备份。

## MCP 的作用

如有远程 MCP 服务，可以让 MCP 调用发布脚本或上传接口，自动完成发布；安装包本身仍建议通过 HTTP/HTTPS 或对象存储分发，避免大文件占用 MCP 对话通道。
