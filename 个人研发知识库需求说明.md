# 个人研发知识库需求说明

## 一、描述

建设一个运行在本地电脑上的个人研发知识库。

知识库以真实的文件夹和 Markdown 文件为核心，不依赖在线服务。网页主要提供内容定位、阅读、编辑和 AI 排版能力。所有修改最终写回原始 Markdown 文件，不建立重复的内容数据库。

知识库顶部栏目为：

```text
首页｜任务｜项目｜文献｜报告｜SOP｜软件｜写作
```

目录与网页直接对应：

- 最外层文件夹对应顶部栏目。
- 栏目内的文件夹和子文件夹对应左侧多级导航。
- Markdown 文件对应左侧可点击页面。
- Markdown 内容显示在页面中间。
- 当前 Markdown 的标题层级自动生成右侧目录。
- 图片、PDF、表格等附件存放在所属内容附近。

首页保持简洁，只展示 Logo、知识库简介和快速开始入口，不设计成项目看板。

## 二、功能

### 1. 文件夹自动映射

网页自动读取知识库目录，不需要手工维护导航配置。

```text
knowledge_base\
├── 任务\
├── 项目\
├── 文献\
├── 报告\
├── SOP\
├── 软件\
└── 写作\
```

新增、删除、移动或重命名文件夹和 Markdown 文件后，顶部导航及左侧目录自动更新。

### 2. 多级内容导航

- 顶部切换一级工作内容。
- 左侧展示当前栏目下的多层文件夹。
- 所有文件夹均可展开或折叠。
- Markdown 文件作为末级页面显示。
- 不强制项目内部使用固定模板。

### 3. Markdown 阅读

中间区域展示 Markdown 最终效果，包括：

- 多级标题和段落
- 列表、表格和引用
- 代码块及提示框
- 图片、PDF 和附件链接
- 内部文档链接

右侧根据当前文档标题自动生成目录，并随阅读位置高亮。

### 4. 阅读与编辑切换

每个 Markdown 页面提供：

```text
阅读模式｜编辑模式
```

阅读模式用于干净展示；编辑模式采用所见即所得方式，编辑时的排版应尽量与最终阅读效果保持一致。

保存后直接写回原 `.md` 文件。保存前检测外部修改，避免与 Codex、Claude Code 或其他编辑器产生覆盖冲突。

### 5. 文件管理

支持在对应目录层级下：

- 新建文件和文件夹
- 重命名及移动
- 删除前确认
- 打开文件所在位置
- 使用系统编辑器打开
- 显示文件路径和修改时间

### 6. 附件管理

附件不集中存放，而是跟随对应项目或文档：

```text
项目\新橙皮苷\酶筛选\
├── 筛选结果.md
└── assets\
    ├── 活性对比图.png
    └── 原始数据.xlsx
```

支持拖入图片，并使用相对路径写入 Markdown。项目整体移动或迁移后，附件引用仍然有效。

### 7. 检索与阅读设置

- 全局搜索文件名和正文
- 当前栏目范围搜索
- 复制页面内容或文件路径
- 深色与浅色模式
- 字号、行距和内容宽度调整
- 上一篇和下一篇
- 最近打开记录

### 8. AI 排版 Agent

提供独立的“AI 排版”按钮，用于处理当前 Markdown：

- 调整标题层级
- 整理段落、列表、表格和引用
- 统一 Markdown 格式
- 修复多余空行和错误缩进
- 检查图片相对路径
- 提示过长段落、重复标题和结构问题

执行过程为：

```text
当前文档
→ AI 生成排版建议
→ 展示修改前后差异
→ 用户确认
→ 写回 Markdown
→ 保存历史版本
```

Agent 不得修改实验数据、参数、事实和结论，不得编造内容，也不得未经确认直接覆盖文件。

## 三、技术亮点

### 1. 文件即知识库

不使用数据库保存正文。文件夹负责分类，Markdown 负责内容，附件就近存放。知识库能够直接复制、备份和迁移。

### 2. 导航零配置

根据文件系统实时生成：

```text
一级文件夹 → 顶部导航
多级子文件夹 → 左侧导航树
Markdown 标题 → 右侧页面目录
```

不需要同时维护文件目录和网站配置。

### 3. 所见即所得且保留 Markdown

网页提供接近最终页面效果的编辑体验，同时以标准 Markdown 作为保存格式，不绑定某个专有编辑软件。

### 4. 多工具共享同一内容

Codex、Claude Code、Obsidian、VS Code 及本地网页可以同时读取同一个知识库目录，不需要导入、导出或同步副本。

### 5. 本地优先

网站、文件和 Agent 均可在本地运行。公司内部资料无需上传互联网，并可通过私有配置决定允许哪种模型读取哪些目录。

### 6. 安全写入

- 保存前检测文件版本
- 修改前后差异对比
- 自动生成历史版本
- 删除和覆盖需要确认
- 限制 Agent 只能处理当前文档

### 7. 可替换的 AI 接口

AI 排版功能与模型分离，可接入：

- Codex CLI
- Claude Code CLI
- DeepSeek API 或本地兼容接口
- 后续其他模型

更换模型不会影响文件目录和 Markdown 内容。

### 8. 自动更新

编辑或新增 Markdown 后，网页自动重新加载内容；目录变化时，顶部和左侧导航同步更新，无需重新构建整个知识库。
